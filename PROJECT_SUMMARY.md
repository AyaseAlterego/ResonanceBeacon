## 架构亮点

### 设计模式
1. **基于类别的智能体路由**：根据任务类别自动选择最佳智能体
2. **工厂 + 依赖注入模式**：所有组件通过工厂函数创建，支持测试注入
3. **安全创建模式**：包装工厂创建，确保组件失败不会崩溃系统
4. **多层级配置系统**：配置优先级（项目级 > 用户级 > 默认值）
5. **五层钩子系统**：将系统生命周期钩子组织为 5 个层级
6. **后台任务管理器**：借鉴 oh-my-openagent 的 BackgroundManager

### 容错性
- 安全创建：一个组件失败不会崩溃整个系统
- 熔断器：自动检测和恢复故障
- 并发管理器：防止系统过载
- 任务重试：自动重试失败任务
- 循环检测：防止无限循环

## 使用指南

### 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/AyaseAlterego/ResonanceBeacon.git
cd ResonanceBeacon

# 2. 安装依赖
pip install -e ".[dev]"

# 3. 启动服务
docker-compose up -d

# 4. 运行数据库迁移
alembic upgrade head

# 5. 测试 CLI
hermes --help

# 6. 运行测试
pytest tests/ -v
```

### 运行第一个流水线

```bash
# 创建项目
hermes 项目 创建 --名称 "我的项目"

# 运行流水线
hermes 流水线 运行 流水线定义.json --输入 "构建 REST API"

# 查看状态
hermes 流水线 状态 <流水线ID>
```
