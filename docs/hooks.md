# 五层钩子系统

起源信标 (ResonanceBeacon) 的钩子系统将流水线生命周期中的各类事件组织为五个层级，每层负责不同的关注点。钩子通过工厂模式创建、由组合器统一组装，并遵循安全创建原则——单个钩子失败不会崩溃整个系统。

## 架构概述

```
┌─────────────────────────────────────────────────┐
│                  钩子组合器                        │
│              (统一组装与触发入口)                    │
├────────┬────────┬────────┬────────┬──────────────┤
│ 会话层  │工具守护层│ 转换层  │ 继续层  │   技能层     │
│Session │Guard   │Transform│Continue│   Skill     │
├────────┼────────┼────────┼────────┼──────────────┤
│生命周期 │验证/权限 │数据变换  │重试/循环 │技能检测/评估  │
│事件通知 │日志记录 │上下文传递 │失败恢复 │需求匹配      │
└────────┴────────┴────────┴────────┴──────────────┘
```

**设计原则：**

- **工厂模式**：每层钩子由独立的工厂类创建，便于替换和测试
- **安全创建**：所有钩子通过 `安全创建钩子` 包装创建，单个钩子失败返回 `None` 而不抛出异常
- **优先级排序**：同层钩子按优先级（0–10，10最高）降序执行
- **容错执行**：触发钩子时单个回调异常被捕获并记录日志，不影响其他钩子继续执行

## 核心定义

> 源码：`src/hermes/钩子/基础.py`

### 钩子层级 (钩子层级)

```python
class 钩子层级(Enum):
    会话 = "session"           # 生命周期事件，恢复，通知
    工具守护 = "tool_guard"     # 工具执行前后守护
    转换 = "transform"         # 消息/上下文变换
    继续 = "continuation"       # 循环管理，重试
    技能 = "skill"              # 技能生命周期
```

### 钩子事件 (钩子事件)

```python
class 钩子事件(Enum):
    # 流水线事件
    流水线已启动 = "pipeline_started"
    流水线已完成 = "pipeline_completed"
    流水线已失败 = "pipeline_failed"
    流水线已取消 = "pipeline_cancelled"

    # 阶段事件
    阶段已开始 = "stage_started"
    阶段已完成 = "stage_completed"
    阶段已失败 = "stage_failed"

    # 任务事件
    任务已开始 = "task_started"
    任务已完成 = "task_completed"
    任务已失败 = "task_failed"
    任务已重试 = "task_retried"

    # 审批事件
    审批已请求 = "approval_requested"
    审批已批准 = "approval_approved"
    审批已拒绝 = "approval_rejected"

    # 智能体事件
    智能体已选择 = "agent_selected"
    智能体执行开始 = "agent_execution_started"
    智能体执行完成 = "agent_execution_completed"
    智能体执行失败 = "agent_execution_failed"

    # 错误事件
    错误已发生 = "error_occurred"
    错误已恢复 = "error_recovered"
```

### 钩子数据类 (钩子)

```python
@dataclass
class 钩子:
    名称: str                              # 钩子唯一标识名
    层级: 钩子层级                          # 所属层级
    事件: 钩子事件                          # 监听的事件
    回调: Callable[..., Any]               # 回调函数
    优先级: int = 0                         # 0-10，10最高
    描述: str = ""                          # 钩子用途描述
```

### 钩子上下文 (钩子上下文)

```python
@dataclass
class 钩子上下文:
    流水线: Any = None          # 流水线对象
    阶段: Any = None            # 阶段对象
    任务: Any = None            # 任务对象
    智能体: Any = None          # 智能体对象
    结果: Any = None            # 执行结果
    错误: Optional[Exception] = None  # 错误信息
    元数据: dict[str, Any] = field(default_factory=dict)
```

钩子上下文在工厂创建钩子时传入，使工厂可以根据当前运行环境动态决定是否创建某个钩子或调整其行为。

### 安全创建钩子 (安全创建钩子)

```python
def 安全创建钩子(
    名称: str,
    工厂: Callable[[钩子上下文], Optional[钩子]],
    上下文: 钩子上下文
) -> Optional[钩子]:
```

**核心特性：**

1. 捕获工厂函数中的所有异常，返回 `None` 而不是崩溃
2. 记录详细的错误日志（含堆栈跟踪）
3. 实现优雅降级——一个钩子创建失败不影响其他钩子

创建失败的钩子（返回 `None` 的）会在工厂的 `创建钩子` 方法中被过滤掉：

```python
钩子列表 = [h for h in 钩子列表 if h is not None]
```

## 钩子组合器

> 源码：`src/hermes/钩子/组合器.py`

`钩子组合器` 是钩子系统的统一入口，负责组装五层钩子并触发事件。

### 初始化

组合器内部持有五个工厂实例，按层级映射：

```python
class 钩子组合器:
    def __init__(self):
        self.钩子工厂 = {
            钩子层级.会话: 会话钩子工厂(),
            钩子层级.工具守护: 工具守护钩子工厂(),
            钩子层级.转换: 转换钩子工厂(),
            钩子层级.继续: 继续钩子工厂(),
            钩子层级.技能: 技能钩子工厂(),
        }
```

### 组合方法

```python
def 组合(self, 上下文: 钩子上下文) -> dict[钩子层级, list[钩子]]:
```

遍历五个工厂，依次调用各工厂的 `创建钩子(上下文)` 方法，返回按层级组织的钩子字典。如果某个层级的工厂创建失败，该层级会得到一个空列表而非报错。

### 触发方法

```python
def 触发钩子(
    self,
    钩子字典: dict[钩子层级, list[钩子]],
    事件: 钩子事件,
    *参数
):
```

**执行流程：**

1. 遍历所有层级的钩子列表
2. 在每个层级内，按优先级降序排序（高优先级先执行）
3. 匹配事件类型——只调用 `钩子.事件 == 事件` 的钩子
4. 执行回调，捕获并记录异常，不向上传播

### 辅助方法

```python
def 获取钩子数量(self, 钩子字典: dict[钩子层级, list[钩子]]) -> int:
```

返回所有层级的钩子总数。当前系统默认创建 19 个钩子（7 + 4 + 3 + 3 + 2）。

## 五层钩子详解

### 第一层：会话层

> 源码：`src/hermes/钩子/会话.py` — `会话钩子工厂`

**职责：** 记录流水线、阶段、任务的生命周期事件通知。

| 钩子名称 | 监听事件 | 优先级 | 回调参数 | 描述 |
|---------|---------|-------|---------|------|
| 流水线启动通知 | `流水线已启动` | 5 | `(流水线)` | 记录流水线启动 |
| 流水线完成通知 | `流水线已完成` | 5 | `(流水线, 结果)` | 记录流水线完成 |
| 流水线失败通知 | `流水线已失败` | 10 | `(流水线, 错误)` | 记录流水线失败（高优先级） |
| 阶段开始通知 | `阶段已开始` | 3 | `(阶段)` | 记录阶段开始 |
| 阶段完成通知 | `阶段已完成` | 3 | `(阶段)` | 记录阶段完成 |
| 任务开始通知 | `任务已开始` | 2 | `(任务)` | 记录任务开始 |
| 任务完成通知 | `任务已完成` | 2 | `(任务, 结果)` | 记录任务完成 |

**触发时机：** 流水线引擎在状态转换时触发对应事件。

**回调特性：** 回调函数兼容 `dict` 和对象两种数据格式——优先从 dict 取值，其次从对象属性取值，最后使用 `"未知"` 默认值。

**典型用途：**

- 向外部系统（如 Slack、邮件）发送流水线状态通知
- 记录审计日志
- 触发监控告警（流水线失败时优先级最高）

---

### 第二层：工具守护层

> 源码：`src/hermes/钩子/工具守护/工具守护钩子.py` — `工具守护钩子工厂`

**职责：** 在任务执行前后进行验证、权限检查和结果记录。

| 钩子名称 | 监听事件 | 优先级 | 回调参数 | 返回值 | 描述 |
|---------|---------|-------|---------|-------|------|
| 任务输入验证 | `任务已开始` | 10 | `(任务定义)` | `True`/`False` | 验证任务定义包含 `id`、`name`、`type` 必要字段 |
| 任务权限检查 | `任务已开始` | 9 | `(任务定义)` | `True` | 检查任务执行权限（当前允许所有类别） |
| 任务结果记录 | `任务已完成` | 5 | `(任务定义, 结果)` | 无 | 记录任务完成状态，区分成功/未成功 |
| 任务错误处理 | `任务已失败` | 8 | `(任务定义, 错误)` | 无 | 记录错误日志，预留恢复逻辑扩展点 |

**触发时机：** 任务开始前触发验证和权限检查；任务完成后记录结果；任务失败时处理错误。

**典型用途：**

- 输入数据校验——拒绝不符合规范的任务定义
- 权限控制——根据类别或用户角色限制任务执行
- 审计追踪——记录每个任务的执行结果
- 错误恢复——捕获异常并尝试自动修复

---

### 第三层：转换层

> 源码：`src/hermes/钩子/转换/转换钩子.py` — `转换钩子工厂`

**职责：** 在阶段和任务之间进行数据转换与上下文传递。

| 钩子名称 | 监听事件 | 优先级 | 回调参数 | 描述 |
|---------|---------|-------|---------|------|
| 阶段结果转换 | `阶段已完成` | 5 | `(阶段定义)` | 将阶段输出转换为下一阶段的输入格式 |
| 上下文数据合并 | `阶段已完成` | 4 | `(阶段定义)` | 合并阶段数据到流水线上下文中 |
| 任务输出格式化 | `任务已完成` | 3 | `(任务定义, 结果)` | 将原始输出转换为结构化格式 |

**触发时机：** 阶段完成后立即触发数据转换和上下文合并；任务完成后格式化输出。

**执行顺序：** 同为 `阶段已完成` 事件时，阶段结果转换（优先级 5）先于上下文数据合并（优先级 4）执行，确保数据先转换再合并。

**典型用途：**

- 将需求分析阶段的自然语言输出转换为结构化 JSON
- 将设计文档转换为开发任务的输入参数
- 标准化不同智能体的输出格式

---

### 第四层：继续层

> 源码：`src/hermes/钩子/继续/继续钩子.py` — `继续钩子工厂`

**职责：** 管理流程的继续/中断决策，包括重试、循环检测和失败恢复。

| 钩子名称 | 监听事件 | 优先级 | 回调参数 | 返回值 | 描述 |
|---------|---------|-------|---------|-------|------|
| 任务失败重试 | `任务已失败` | 10 | `(任务定义, 错误)` | `True`（应重试）/ `False`（不重试） | 默认最多重试 3 次 |
| 无限循环检测 | `任务已开始` | 15 | `(任务定义)` | `True`（可继续）/ `False`（应中止） | 检查任务是否依赖于自身 |
| 阶段失败恢复 | `阶段已失败` | 8 | `(阶段定义)` | 无 | 预留阶段级恢复逻辑 |

**触发时机：** 任务开始前检测循环依赖；任务失败时决定是否重试；阶段失败时尝试恢复。

**优先级说明：** 无限循环检测的优先级为 15（超出 0–10 的常规范围），确保它在所有 `任务已开始` 钩子中最先执行——循环检测必须先于其他任何操作。

**典型用途：**

- 自动重试因网络波动等临时性原因失败的任务
- 防止 DAG 中出现循环依赖导致死循环
- 阶段失败时执行降级策略（如跳过非关键阶段）

---

### 第五层：技能层

> 源码：`src/hermes/钩子/技能/技能钩子.py` — `技能钩子工厂`

**职责：** 检测任务所需技能并评估技能执行质量。

| 钩子名称 | 监听事件 | 优先级 | 回调参数 | 返回值 | 描述 |
|---------|---------|-------|---------|-------|------|
| 技能需求检测 | `任务已开始` | 5 | `(任务定义)` | 技能名称列表 | 根据任务类型匹配技能（如 `code_generation` → `"代码生成"`） |
| 技能执行评估 | `任务已完成` | 5 | `(任务定义, 结果)` | `True`/`False` | 评估技能执行是否成功 |

**触发时机：** 任务开始时检测需要哪些技能；任务完成时评估技能执行质量。

**技能映射规则：**

| 任务类型 | 匹配技能 |
|---------|---------|
| `code_generation` | 代码生成 |
| `code_review` | 代码审查 |
| `testing` | 测试生成 |
| 其他 | 无 |

**典型用途：**

- 根据任务类型自动加载对应的技能模块
- 评估 AI 智能体执行结果是否满足技能质量标准
- 为后续任务调度提供技能匹配依据

## 执行顺序与生命周期

### 流水线完整生命周期中的钩子触发时序

```
  ▼

│                                  │

│  │    任务权限检查 (优先级9)    │   │

│  │    技能需求检测 (优先级5)    │   │
│  │  → 会话层:                 │   │
│  │    任务开始通知 (优先级2)    │   │
│  │                            │   │
│  │  [任务执行...]              │   │
│  │                            │   │
│  │  成功时:                    │   │

│  │  → 工具守护层:             │   │

│  │    技能执行评估 (优先级5)    │   │
│  │  → 会话层:                 │   │
│  │    任务完成通知 (优先级2)    │   │

│  │  → 继续层:                 │   │
│  │    任务失败重试 (优先级10)   │   │
│  │  → 工具守护层:             │   │
│  │    任务错误处理 (优先级8)    │   │
│  │                            │   │

│                                  │
│  触发: 阶段已完成                 │
│  → 转换层:                       │
│    阶段结果转换 (优先级5)          │
│    上下文数据合并 (优先级4)        │
│  → 会话层:                       │
│    阶段完成通知 (优先级3)          │
│                                  │
│  或: 阶段已失败                   │

│ 触发: 流水线已完成                │

│ 触发: 流水线已失败                │

```

### 同一事件下的跨层执行顺序

当多个层级的钩子监听同一事件时，组合器按层级遍历顺序依次执行（字典迭代顺序）。同一层级内按优先级降序执行。

以 `任务已开始` 为例，涉及四个层级共 4 个钩子：

| 层级 | 钩子 | 优先级 | 执行位置 |
|------|------|-------|---------|
| 工具守护 | 任务输入验证 | 10 | 第1层内最高 |
| 工具守护 | 任务权限检查 | 9 | 第1层级内第二 |
| 继续 | 无限循环检测 | 15 | 第4层内最高 |
| 技能 | 技能需求检测 | 5 | 第5层内唯一 |
| 会话 | 任务开始通知 | 2 | 第1层内最低 |

### 容错机制

- **创建阶段**：`安全创建钩子` 捕获异常，返回 `None`，过滤后不影响其他钩子
- **执行阶段**：`触发钩子` 中每个回调的异常被捕获并记录日志，不向上传播
- **层级级别**：单个层级的工厂创建失败时，该层级得到空列表，不影响其他层级

## 编写自定义钩子

### 第一步：定义钩子工厂方法

在对应层级的工厂类中添加一个 `_创建Xxx` 方法，遵循以下模式：

```python
def _创建自定义通知(self, 上下文: 钩子上下文) -> Optional[钩子]:
    """创建自定义通知钩子"""
    def 回调(流水线):
        # 自定义逻辑
        名称 = 流水线.get("name", "未知") if isinstance(流水线, dict) else (
            流水线.名称 if hasattr(流水线, "名称") else "未知"
        )
        logger.info(f"自定义通知: {名称}")

    return 钩子(
        名称="自定义通知",
        层级=钩子层级.会话,        # 指定所属层级
        事件=钩子事件.流水线已启动,  # 指定监听事件
        回调=回调,
        优先级=7,                   # 设置优先级
        描述="自定义的流水线启动通知"
    )
```

### 第二步：注册到工厂的创建钩子方法

将新钩子通过 `安全创建钩子` 包装后添加到 `创建钩子` 的返回列表中：

```python
def 创建钩子(self, 上下文: 钩子上下文) -> list[钩子]:
    钩子列表 = [
        # ... 已有钩子 ...
        安全创建钩子(
            "自定义通知",
            self._创建自定义通知,
            上下文
        ),
    ]
    return [h for h in 钩子列表 if h is not None]
```

### 第三步（可选）：创建独立的工厂类

如果自定义钩子逻辑较复杂，建议创建独立的工厂类：

```python
# src/hermes/钩子/自定义/自定义钩子.py

from typing import Optional
import logging
from ..基础 import 钩子, 钩子层级, 钩子事件, 钩子上下文, 安全创建钩子

logger = logging.getLogger(__name__)

class 自定义钩子工厂:
    """自定义层钩子工厂"""

    def 创建钩子(self, 上下文: 钩子上下文) -> list[钩子]:
        钩子列表 = [
            安全创建钩子(
                "Slack通知",
                self._创建Slack通知,
                上下文
            ),
            安全创建钩子(
                "指标采集",
                self._创建指标采集,
                上下文
            ),
        ]
        return [h for h in 钩子列表 if h is not None]

    def _创建Slack通知(self, 上下文: 钩子上下文) -> Optional[钩子]:
        def 回调(流水线, 错误):
            名称 = 流水线.get("name", "未知") if isinstance(流水线, dict) else "未知"
            # 发送Slack通知（示例）
            logger.info(f"[Slack] 流水线失败: {名称}, 错误: {错误}")

        return 钩子(
            名称="Slack通知",
            层级=钩子层级.会话,
            事件=钩子事件.流水线已失败,
            回调=回调,
            优先级=10,
            描述="流水线失败时发送Slack通知"
        )

    def _创建指标采集(self, 上下文: 钩子上下文) -> Optional[钩子]:
        def 回调(任务定义, 结果):
            任务ID = 任务定义.get("id", "未知") if isinstance(任务定义, dict) else "未知"
            # 采集指标（示例）
            logger.info(f"[指标] 任务 {任务ID} 完成")

        return 钩子(
            名称="指标采集",
            层级=钩子层级.工具守护,
            事件=钩子事件.任务已完成,
            回调=回调,
            优先级=1,
            描述="采集任务完成指标"
        )
```

### 第四步：注册到组合器

在 `钩子组合器.__init__` 中添加新的工厂实例（如需新增层级），或在已有工厂类中直接添加方法。

如果创建了新的层级枚举值，需要同时更新 `钩子层级` 枚举。

### 编写注意事项

1. **回调兼容性**：回调参数应同时支持 `dict` 和对象两种数据格式，参考会话层钩子的写法
2. **安全创建**：始终通过 `安全创建钩子` 包装工厂方法，确保异常不向上传播
3. **优先级设计**：守护类钩子（验证、权限）应设置高优先级（8–10），通知类钩子设置低优先级（1–5）
4. **幂等性**：回调可能被多次触发，应确保逻辑幂等
5. **返回值约定**：守护类和继续类钩子通过返回 `True`/`False` 控制流程；通知类和记录类钩子无返回值

## 使用场景示例

### 场景一：启动流水线并监听全生命周期

```python
from src.hermes.钩子 import 钩子组合器, 钩子上下文, 钩子事件

# 创建组合器和上下文
组合器 = 钩子组合器()
上下文 = 钩子上下文(流水线={"name": "REST API开发"})

# 组装所有层级的钩子
钩子字典 = 组合器.组合(上下文)

# 触发流水线启动事件
组合器.触发钩子(钩子字典, 钩子事件.流水线已启动, {"name": "REST API开发"})

# 触发阶段事件
组合器.触发钩子(钩子字典, 钩子事件.阶段已开始, {"id": "requirements", "name": "需求分析"})

# 触发任务事件——会自动执行验证、权限检查、循环检测、技能检测
组合器.触发钩子(钩子字典, 钩子事件.任务已开始, {
    "id": "analyze",
    "name": "分析需求",
    "type": "requirements_engineering",
    "category": "exploration"
})

# 任务完成——会自动执行结果记录、输出格式化、技能评估
组合器.触发钩子(钩子字典, 钩子事件.任务已完成, {"id": "analyze"}, {"状态": "completed"})

# 流水线完成
组合器.触发钩子(钩子字典, 钩子事件.流水线已完成, {"name": "REST API开发"}, {"状态": "success"})
```

### 场景二：任务失败时自动重试

```python
# 组装钩子
组合器 = 钩子组合器()
钩子字典 = 组合器.组合(钩子上下文(流水线={}))

# 模拟任务失败
任务定义 = {"id": "backend", "name": "后端开发", "type": "code_generation", "重试次数": 0}
错误 = Exception("网络超时")

# 触发失败事件——继续层的"任务失败重试"钩子会判定是否重试
组合器.触发钩子(钩子字典, 钩子事件.任务已失败, 任务定义, 错误)
# 日志输出: 任务 backend 失败，准备重试 (1/3)

# 第二次失败
任务定义["重试次数"] = 2
组合器.触发钩子(钩子字典, 钩子事件.任务已失败, 任务定义, 错误)
# 日志输出: 任务 backend 失败，准备重试 (3/3)

# 达到最大重试
任务定义["重试次数"] = 3
组合器.触发钩子(钩子字典, 钩子事件.任务已失败, 任务定义, 错误)
# 日志输出: 任务 backend 已达到最大重试次数 3
```
