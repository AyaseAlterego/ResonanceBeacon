## 架构概述

配置系统由三个核心模块组成：

| 模块 | 文件 | 职责 |
|------|------|------|
| **模式** | `src/hermes/配置/模式.py` | 基于 Pydantic 的配置模式定义，包含所有配置项的类型、默认值和验证规则 |
| **合并** | `src/hermes/配置/合并.py` | 安全深度合并实现，防原型污染，支持多层合并 |
| **配置加载器** | `src/hermes/配置/配置加载器.py` | 多层级配置加载，closest-wins 策略，环境变量覆盖，缓存管理 |

外部入口通过全局函数 `获取配置()` 获取合并后的完整配置对象。

## 配置层级与优先级

系统采用 **closest-wins** 策略：越接近项目的配置越优先。从低到高，优先级如下：

```
优先级（低 → 高）

1. 默认值          代码内硬编码的默认值（应用配置()）
2. 系统级配置      /etc/hermes/配置.jsonc
3. 用户级配置      ~/.hermes/配置.jsonc
4. 父目录配置      <父目录>/.hermes/配置.jsonc（沿目录树向上）
5. 项目级配置      <项目>/.hermes/配置.jsonc（最接近当前目录）
6. 环境变量覆盖    HERMES_* 环境变量（最高优先级）
```

### 项目级配置的遍历逻辑

加载器从当前工作目录向上遍历到 HOME 目录，收集所有遇到的 `.hermes/配置.jsonc` 文件，然后反转列表使最接近项目的配置最后合并（从而覆盖更上层的配置）。这意味着如果你在 `/home/user/projects/myapp` 下工作，且以下位置都存在配置文件：

- `/home/user/.hermes/配置.jsonc`
- `/home/user/projects/.hermes/配置.jsonc`
- `/home/user/projects/myapp/.hermes/配置.jsonc`

则 `myapp` 下的配置具有最高优先级。

## 配置文件格式（JSONC）

配置文件使用 **JSONC** 格式，即支持注释的 JSON。文件名为 `配置.jsonc`，放置在 `.hermes` 目录下。

### 支持的注释语法

| 语法 | 说明 |
|------|------|
| `// 注释` | 单行注释（行末注释同样支持） |
| `# 注释` | 井号注释（仅在行内不含引号时生效） |

> **注意**：字符串中的 `//` 和 `#` 不会被误判为注释。

### JSONC 示例

```jsonc
{
  // 项目基本信息
  "项目名称": "我的项目",
  "环境": "development",

  // 智能体配置
  "智能体": {
    "claude_code": {
      "模型": "claude-3-5-sonnet-20241022",  // 使用 Sonnet 模型
      "最大令牌数": 4096,
      "温度": 0.3
    },
    "opencode": {
      "模型": "gpt-4o",
      "温度": 0.5
    }
  },

  # 类别路由配置
  "类别": {
    "ultrabrain": {
      "描述": "架构级，高难度逻辑",
      "智能体优先级": ["claude_code"]
    }
  },

  "流水线默认值": {
    "最大重试次数": 3,
    "任务超时": 600
  }
}
```

### 智能体配置

| 配置项 | 类型 | 默认值 | 约束 | 说明 |

## 安全深度合并机制

配置合并采用安全深度合并算法，防止原型污染攻击。

### 核心函数

| 函数 | 说明 |
|------|------|
| `安全深度合并(基础, 覆盖)` | 合并两个字典，覆盖层的值优先 |
| `安全深度合并多层(层级列表)` | 依次合并多个字典层级（从最不具体到最具体） |
| `检查原型污染(数据)` | 递归检查字典中是否存在危险键 |

### 合并规则

| 数据类型 | 合并策略 | 示例 |
|---------|---------|------|
| **嵌套字典** | 递归合并 | `{"a": {"b": 1}}` + `{"a": {"c": 2}}` → `{"a": {"b": 1, "c": 2}}` |
| **列表** | 集合并集（去重） | `[1, 2, 3]` + `[3, 4, 5]` → `[1, 2, 3, 4, 5]` |
| **标量值** | 直接覆盖 | `1` + `2` → `2` |

### 原型污染防护

以下危险键会被阻止，发现时抛出 `ValueError`：

| 危险键 | 阻止原因 |
|--------|---------|
| `__proto__` | 可修改对象原型链 |
| `constructor` | 可访问构造函数 |
| `prototype` | 可修改原型属性 |

```python
# 尝试注入原型污染会抛出异常
安全深度合并({"a": 1}, {"__proto__": {"恶意": "数据"}})
# → ValueError: 阻止潜在危险的键: __proto__
```

### 深度限制

合并操作有最大深度限制（默认 50 层），超过时抛出 `ValueError`，防止循环引用导致无限递归：

```python
# 超过最大深度会抛出异常
安全深度合并(极深字典, 覆盖)  # 深度 > 50
# → ValueError: 合并深度超过最大限制 (50)，可能存在循环引用
```

### 原型污染检查

`检查原型污染` 函数会递归扫描整个字典，返回所有危险键的路径列表：

```python
数据 = {
    "a": 1,
    "__proto__": {"恶意": "数据"},
    "b": {"constructor": "危险"}
}
危险键 = 检查原型污染(数据)
# → ["__proto__", "b.constructor"]
```

在配置文件加载时，系统会自动调用此函数进行预检查，发现危险键时记录警告日志。

## 环境变量覆盖

环境变量具有最高优先级，可以覆盖任何配置文件中的设置。环境变量前缀为 `HERMES_`。

### 直接映射的环境变量

| 环境变量 | 对应配置项 | 示例值 |
|---------|-----------|--------|
| `HERMES_环境` | `环境` | `production` |
| `HERMES_数据库URL` | `数据库URL` | `postgresql://user:pass@db:5432/prod` |
| `HERMES_REDISURL` | `RedisURL` | `redis://redis:6379/0` |

### 智能体配置的环境变量

格式：`HERMES_智能体_<智能体ID>_<字段名>=<值>`

| 环境变量 | 说明 |
|---------|------|
| `HERMES_智能体_claude_code_模型` | 覆盖 claude_code 智能体的模型 |
| `HERMES_智能体_claude_code_温度` | 覆盖 claude_code 智能体的温度 |
| `HERMES_智能体_opencode_最大令牌数` | 覆盖 opencode 智能体的最大令牌数 |

示例：

```bash
# 设置环境
export HERMES_环境=production

# 设置数据库连接
export HERMES_数据库URL="postgresql+asyncpg://hermes:secure_password@db-server:5432/hermes_prod"

# 覆盖智能体模型
export HERMES_智能体_claude_code_模型="claude-3-opus-20240229"
export HERMES_智能体_claude_code_温度="0.1"
```

## 配置验证规则

### Pydantic 验证

所有配置值在创建 `应用配置` 对象时由 Pydantic 自动验证：

| 配置项 | 验证规则 |
|--------|---------|
| `智能体配置.温度` | `0 ≤ 值 ≤ 1`（使用 `ge=0, le=1` 约束） |
| 所有 `int` 字段 | 必须为整数 |
| 所有 `float` 字段 | 必须为浮点数或可转换的数值 |
| 所有 `str` 字段 | 必须为字符串 |
| 字典字段 | 键和值类型必须匹配模式定义 |

### CLI 验证命令

通过 `hermes 配置 验证` 命令可验证配置文件的完整性：

- 检查必填字段：`项目名称`、`版本`、`环境`
- 检查环境值：必须是 `development`、`staging`、`production` 之一
- 发现错误时返回非零退出码

## 常见配置场景

### 场景1：开发环境默认配置

无需创建任何配置文件，系统内置的默认值即可满足开发需求。

```jsonc
// ~/.hermes/配置.jsonc（可选，用户级偏好）
{
  "环境": "development",
  "智能体": {
    "claude_code": {
      "模型": "claude-3-5-sonnet-20241022",
      "温度": 0.3
    }
  }
}
```

### 场景2：生产环境部署

通过环境变量覆盖敏感配置，避免将密码写入文件：

```bash
export HERMES_环境=production
export HERMES_数据库URL="postgresql+asyncpg://hermes:secure_password@db-server:5432/hermes_prod"
export HERMES_REDISURL="redis://redis-server:6379/0"
```

配合项目级配置设置更严格的超时和质量阈值：

```jsonc
// <项目>/.hermes/配置.jsonc
{
  "环境": "production",
  "流水线默认值": {
    "最大重试次数": 5,
    "任务超时": 1200,
    "测试覆盖率最低值": 0.90,
    "最大lint错误数": 0,
    "最大安全关键问题数": 0
  },
  "后台任务": {
    "每智能体最大并发数": 10,
    "熔断器阈值": 3,
    "熔断器恢复时间": 120
  }
}
```

### 场景3：自定义智能体和类别

添加新的智能体或调整类别路由：

```jsonc
{
  "智能体": {
    "claude_code": {
      "模型": "claude-3-opus-20240229",
      "最大令牌数": 8192,
      "温度": 0.1,
      "每千令牌成本": 0.015
```
